# 登录 404 修复执行计划

## 定位结论

- cartl 请求前缀与 Vite proxy 前缀不一致：请求使用 `/proxy-default`，配置只覆盖 `/api`。

## 改动点

1. 在 `vite.config.ts` 增加 `/proxy-default` 到 Dagine 后端 `/dashboard` 服务的开发代理，并移除浏览器侧代理前缀。
2. 不修改 `/auth/login` service 路径、请求封装默认路径或后端路由。

## 影响范围

- In scope：cartl 本地开发时所有使用 `/proxy-default` 的请求。
- Out of scope：生产部署网关配置、后端服务可用性、认证业务逻辑。

## 轻量 Gates

| Gate | 结果 | 处理 |
| --- | --- | --- |
| DB schema | n-a | 无表结构变更。 |
| code generation | n-a | 无生成代码。 |
| permission | n-a | 不调整权限。 |
| API contract | n-a | 保持既有 `POST /dashboard/auth/login` 契约。 |
| data repair | n-a | 无数据操作。 |

## 验证

- 使用 Vite 开发服务器请求 `/proxy-default/auth/login`，确认其被转发而非 Vite 404。
- 执行 `pnpm typecheck`、`pnpm test -- --run`、`pnpm build`。

## 回滚

- 回滚 `vite.config.ts` 中新增的 `/proxy-default` proxy 项即可恢复改动前配置。
