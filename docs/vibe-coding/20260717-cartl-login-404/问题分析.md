# 登录 404 问题分析

## 输入与范围

- 来源：用户反馈 cartl 登录时提示 404。
- 范围：仅修复 `cartl` 本地 Vite 开发代理；不修改后端、Vue 源工程或认证接口契约。

## 现象与复现路径

- 登录页调用 `POST /auth/login`。
- 请求封装默认基础路径是 `/proxy-default`，浏览器实际请求为 `POST /proxy-default/auth/login`。
- cartl 的 Vite 配置此前仅代理 `/api`，未代理 `/proxy-default`，请求由 Vite 开发服务器处理并返回 404。

## 根因与证据

- `src/utils/request/index.ts` 将默认 `baseURL` 设置为 `/proxy-default`。
- `vite.config.ts` 缺失 `/proxy-default` 代理项。
- 原 Vue 工程开发环境将 `/proxy-default` 代理到 `http://localhost:8080/dashboard`，并移除代理前缀。
- 后端路由注册在 `/dashboard/auth/login`，因此 service 中的 `/auth/login` 路径正确。

## 修复目标与验证标准

- Vite 将 `/proxy-default/auth/login` 转发为 `http://127.0.0.1:8080/dashboard/auth/login`。
- 保留现有 `/api` 代理，避免影响无关调用。
- 类型检查、现有测试和生产构建通过；本地代理请求不再由 Vite 返回 404。

## 规模与风险

- 规模：S（单个前端开发配置问题，无 API、权限、数据或数据库变更）。
- 风险：后端未启动时会出现连接失败或后端业务响应，这不等同于此前 Vite 的 404。
