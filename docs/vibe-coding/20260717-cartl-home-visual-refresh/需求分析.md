# Cartl 首页视觉重构需求分析

## 输入来源
- 用户需求：遵循 fullstack-flow React 规范，在 `cartl` 中实现，并以原 Vue 首页 `http://localhost:9527/home` 为基础优化样式。
- 参考实现：`dagine-dashboard/web` 的 Vue 首页、导航、主题、路由与 API 契约。
- 目标项目：`/Users/lucas/workspace/code/golang/cartl`。

## 当前状态
- 需求属于前端 React 重构，规模判定为 M。
- 已确认 React 19、React Router 7、Vite、React Flow、Recharts、Lucide 技术栈。
- 本轮聚焦首页、应用框架和已有数据流页面的视觉一致性，不修改原 Vue 工程和后端。

## 功能范围
- 以原 Vue 首页的信息密度、导航层级、卡片尺寸和图表布局为视觉基线。
- 保留 React 实现，优化左侧导航、顶部栏、首页指标、趋势、动态、数据源健康度和质量概览。
- 完善桌面端与移动端可用性，修复小屏导航覆盖内容的问题。
- 按 React 规范拆分路由、布局、页面和公共组件，避免根组件继续膨胀。

## 非目标
- 不修改 `dagine-dashboard/web`。
- 不修改后端 API 契约、数据库或鉴权模型。
- 本轮不承诺将全部旧 Vue 业务表单和详情页完成真实 API 联调。

## 验收标准
1. 桌面首页在 1440px 视口下与原 Vue 首页具有相同的信息结构和专业后台密度。
2. React 首页保持独立视觉优化：更清晰的层级、颜色和状态反馈，不退化为营销页。
3. 移动端可看到完整内容，导航通过遮罩抽屉打开和关闭。
4. 路由切换不整页刷新，首页、列表和流程设计器均可访问。
5. `pnpm typecheck`、`pnpm build` 通过，并完成桌面/移动浏览器验收。

## Gates
| Gate | 结果 | 处理 |
|---|---|---|
| DB schema | n-a | 无数据库变更 |
| code generation | n-a | 无生成代码 |
| permission | n-a | 沿用现状，未改变权限模型 |
| API contract | no change | 保留现有 `/api/v1` 路径 |

## 假设与风险
- 假设本轮“在原来的基础上”是以原 Vue 的视觉和信息架构为基线，代码仍落到 `cartl` React 项目。
- 当前 `cartl` Git 工作区已有大量用户删除项，本需求只改新增的 React 文件和本需求文档，不恢复或覆盖无关改动。
- 项目尚未配置前端测试框架，关键交互使用 TypeScript、production build 和浏览器检查覆盖。
