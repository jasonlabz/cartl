# Cartl 首页视觉重构技术方案

## 1. 背景与目标
现有 React 原型已覆盖主要导航和流程画布，但视觉密度、字体、桌面信息层级和移动端行为偏离原 Vue 首页。本方案保持 React 技术路线，以原页面作为基线完成视觉与结构重构。

## 2. 范围与非目标
- 范围：应用壳层、首页、资源页、数据流列表、流程设计器的路由与视觉一致性。
- 非目标：Vue 源码修改、后端接口变更、数据库迁移、真实数据写操作。

## 3. 项目规则与技术栈判断
- React 19 + TypeScript strict + Vite。
- React Router 7 使用 `Routes` / `Route`，内部导航统一 `Link` / `NavLink`。
- UI 继续使用项目已安装的轻量组件与 CSS；插件默认 Ant Design/antd-style 不覆盖项目已形成的 Lucide + CSS 体系。
- 样式使用设计 token、组件语义 class 和响应式 media query；卡片圆角不超过 8px。

## 4. 前端设计
### 路由与视图树
- `App`：主题与路由入口。
- `AppLayout`：侧栏、顶部栏、移动端抽屉、命令搜索。
- `HomePage`：原首页信息架构的 React 实现。
- `FlowListPage`、`FlowDesignerPage`、`ResourcePage`：保留现有功能并统一外壳。

### 组件清单
- 通用：`PageHeader`、`Button`、`StatusBadge`。
- 首页：`MetricCard`、`TrendChart`、`ActivityFeed`、`SourceHealthTable`、`QualityOverview`。
- 布局：`Brand`、`Sidebar`、`Topbar`、`CommandPalette`。

### 状态与数据流
- 页面级筛选保留在对应页面内。
- 侧栏和主题状态由 `AppLayout` 管理；移动端路由切换自动关闭抽屉。
- 当前高保真数据来自 `data.ts`；API 层保持独立，后续联调不改变页面结构。

### 边界状态
- 现阶段无真实异步请求，loading/error/permission 不适用；空筛选结果显示明确空态。
- 移动端侧栏为遮罩抽屉，避免内容永久被覆盖。

## 5. 兼容与回滚
- 不触碰 Vue 工程，可直接停止或移除 `cartl` React 项目回滚。
- 页面变更均局限于 `cartl/src`，无存量数据迁移。

## 6. 验证方案
- `pnpm typecheck`
- `pnpm build`
- 浏览器检查 `/home`、`/workflow/flow-designer`、`/workflow/flow-designer/101`。
- 1440x900 与 390x844 视觉检查，验证无空白、重叠和内容遮挡。

## 7. 未决问题
