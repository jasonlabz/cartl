按路径对齐的页面模式

表格页

组织、规则、任务、报告、目录等使用 tableParams；筛选、排序和分页由一次状态更新触发请求。表格区域使用 Spin、Empty 与错误态。

卡片与抽屉

数据源、资产仓库、算子等使用 listParams、Row/Col、独立 Pagination；详情和编辑回填原有 Drawer/Modal 交互。

表单与版本

所有提交字段由 Form.useForm() 管理；弹窗关闭 reset + destroy，复杂标准详情分拆为业务子组件。

工作流的专有边界

持久化模型

  • FlowDocument：节点、边、全局配置、revision
  • WorkbenchState：按节点/目标组织的后端工作台数据
  • 保存、导入、导出、重置均使用原接口 payload

画布 UI 模型

  • React Flow 的节点、边、选中和视口仅保存 UI 状态
  • flow-adapter 负责双向转换、连线和数据校验
  • 避免把 React Flow 状态直接当成后端请求体

服务名称会采用 React 规范的 xxxApi 后缀，但所有 HTTP 方法、URL、参数、响应和错误码保持 Vue 现状。
