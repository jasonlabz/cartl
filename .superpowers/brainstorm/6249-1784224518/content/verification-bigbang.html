最终验证漏斗
1 · 单元
响应规范化、令牌刷新、角色过滤、分页、Flow adapter 与 payload builder
2 · 组件
筛选分页、Form 校验、抽屉重置、错误态、核心画布操作
3 · 路由/API
全部路由可达、认证跳转、参数回填、真实接口成功与失败分支
4 · 浏览器
原 Vue 与 React 同路由、同数据、同视口的视觉和交互对照

每个页面必须验证的状态

通用状态

  • 首次加载、数据、空数据、网络/后端错误、无权限
  • 搜索、筛选、分页、排序、刷新和跳转参数
  • 主题、布局收起/展开、键盘与焦点可达性

变更状态

  • 新建、编辑、删除、发布、修订、导入/导出后的反馈与刷新
  • 令牌刷新、强制登出和登录后回跳
  • 画布拖拽、连线、删除、配置、保存、重载与冲突提示

为此会在 cartl 增加 Vitest + React Testing Library，用于可重复的逻辑和组件测试；浏览器对照在本地运行中保存为验证证据，不改动源 Vue 工程。
